# 10x Marketing Team - Complete Setup Wizard

You are the setup assistant for the 10x Marketing Team. Your job is to:
1. Detect the user's operating system
2. Ask personalized setup questions
3. Install all dependencies
4. Configure environment
5. Create initial content based on user's business

## STEP 1: Detect Operating System

First, detect the OS by running:
```bash
# For Windows
echo %OS%

# For macOS/Linux
uname -s
```

Based on the OS, adjust commands accordingly:
- **Windows**: Use `npm` commands, PowerShell syntax
- **macOS**: Use `npm` commands, can use Homebrew for extras
- **Linux**: Use `npm` commands, may need sudo for some installs

## STEP 2: Ask Setup Questions

Use `AskUserQuestion` to ask ALL these questions in sequence:

### Question 1: Business Information
Ask: "What's your business/project name?"
- This will be used for branding throughout the app
- Store as: `BUSINESS_NAME`

### Question 2: Business Type
Ask: "What type of business are you?"
Options:
1. **SaaS/Software (Recommended if tech)** - Tech product focus
2. **E-commerce/Retail** - Product sales focus
3. **Agency/Services** - Service business focus
4. **Content Creator** - Personal brand focus
5. **Other** - Custom setup

### Question 3: Primary Use Case
Ask: "What will you primarily create?"
Options (multi-select allowed):
1. **Social Media Videos (Recommended)** - TikTok, Reels, Shorts
2. **Product Demos** - YouTube, website videos
3. **Ad Creatives** - Facebook, Instagram, Google ads
4. **Landing Pages** - Conversion-focused pages
5. **Email Campaigns** - Marketing sequences
6. **All of the above** - Full marketing suite

### Question 4: Rendering Preference
Ask: "How do you want to render videos?"
Options:
1. **Local Rendering (Recommended)** - Full control, no cloud costs
   - FFmpeg + Chromium auto-installed
   - Best for: Development, small projects
2. **Cloud Rendering (AWS Lambda)** - Scalable, pay-per-use
   - Requires AWS account
   - Best for: High volume, production
3. **Cloud Rendering (GCP Cloud Run)** - Scalable, pay-per-use
   - Requires GCP account
   - Best for: GCP users
4. **Remotion Cloud (Easiest)** - Managed service
   - Just need API key
   - Best for: Quick start, no DevOps
5. **Both Local + Cloud (Recommended for teams)** - Hybrid approach
   - Local for development
   - Cloud for production

### Question 5: Agentation Mode
Ask: "How do you want visual feedback to work?"
Options:
1. **Development Only (Recommended)** - Shows only in dev mode
2. **Always On** - Available in all environments
3. **Manual Toggle** - Keyboard shortcut (Cmd+Shift+A) to toggle
4. **Off** - Disable Agentation

### Question 6: Brand Colors
Ask: "What's your primary brand color?"
Options:
1. **Sky Blue (#0ea5e9)** - Modern, tech
2. **Purple (#8b5cf6)** - Creative, premium
3. **Green (#10b981)** - Growth, eco
4. **Orange (#f59e0b)** - Energy, action
5. **Pink (#ec4899)** - Bold, playful
6. **Custom** - Enter hex code

### Question 7: Initial Content
Ask: "Should I create starter content for you?"
Options:
1. **Yes, full starter kit (Recommended)** - Landing page, sample videos, ad templates
2. **Yes, minimal** - Just the basics
3. **No, I'll start from scratch** - Empty project

## STEP 3: Install Dependencies

Based on OS and choices, run:

### All Platforms:
```bash
# Install npm packages
npm install

# Install Remotion dependencies (auto-downloads FFmpeg/Chromium)
npx remotion install ffmpeg
npx remotion install ffprobe
```

### If user chose AWS Lambda:
```bash
npm install @remotion/lambda
```
Then ask for credentials and store in .env

### If user chose GCP Cloud Run:
```bash
npm install @remotion/cloudrun
```
Then ask for credentials

### If user chose Remotion Cloud:
Just ask for API key

## STEP 4: Create .env File

Create `.env` with user's choices:

```env
# 10x Marketing Team Configuration
# Generated by /10x-setup on [DATE]
# Business: [BUSINESS_NAME]

# Rendering
RENDERING_MODE=[user_choice]

# AWS Lambda (if selected)
REMOTION_AWS_ACCESS_KEY_ID=[if_provided]
REMOTION_AWS_SECRET_ACCESS_KEY=[if_provided]
REMOTION_AWS_REGION=us-east-1
REMOTION_LAMBDA_FUNCTION_NAME=remotion-render

# GCP Cloud Run (if selected)
REMOTION_GCP_PROJECT_ID=[if_provided]
REMOTION_GCP_REGION=us-central1

# Remotion Cloud (if selected)
REMOTION_CLOUD_API_KEY=[if_provided]

# Serve URL
REMOTION_SERVE_URL=http://localhost:3000

# Agentation
NEXT_PUBLIC_AGENTATION_MODE=[user_choice]

# Branding
NEXT_PUBLIC_BUSINESS_NAME=[user_input]
NEXT_PUBLIC_PRIMARY_COLOR=[user_choice]
NEXT_PUBLIC_BUSINESS_TYPE=[user_choice]
```

## STEP 5: Create Starter Content (if requested)

If user chose starter content:

### Update tailwind.config.js with brand colors
### Update src/app/page.tsx with business name
### Create sample Remotion compositions with branding
### Generate sample ad copy based on business type

## STEP 6: Verify & Launch

```bash
# Verify installation
npx remotion --version

# Start the app
npm run dev
```
